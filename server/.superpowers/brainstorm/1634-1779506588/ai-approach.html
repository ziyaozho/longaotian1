AI 漫画人物绘图 — 实现方案对比

目标：真正的 AI 生成航海王式漫画人物，可根据属性和选择动态变化

核心挑战

AI 图像生成需要 2-10 秒，但选项点击后表情需要即时响应。如何解决"慢生成 vs 快响应"的矛盾？

A

AI 精灵表方案（推荐）

创建角色时，一次性用 AI 生成包含 6 种表情的精灵表。游戏中用 CSS 毫秒级切换，零延迟。

优点

  • 表情切换即时（CSS sprite 切换，<1ms）
  • 每次游戏只调用 1 次 API
  • 可缓存，同一属性组合复用
  • 真正的 AI 画质

缺点

  • 每次新角色需等 3-8 秒生成
  • 表情种类有限（6-8种）
B

实时生成方案

每次表情变化都调用 AI 重新生成。使用 img2img 保持角色一致性。

优点

  • 表情无限多样
  • 装备/伤痕等可精确呈现
  • 每次都独一无二

缺点

  • 每次切换等 3-5 秒
  • API 成本高（每次 $0.01-0.05）
  • 角色一致性难保证
