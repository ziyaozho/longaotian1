## 2026-05-19 修复"回合处理失败"报错

### 根本原因
`narrativeStore.ts` 中 `syncFromPlayer` 方法使用了 `require('../narrative')` 来获取 `getSystemPhase`。浏览器 ESM 环境下不存在 `require`，抛出 `ReferenceError: require is not defined`，被 `startNewTurn` 的 catch 捕获后显示"回合处理失败"。

### 修复内容
1. **`src/store/narrativeStore.ts`**：将 `require` 改为正常的 ESM 顶部导入 `import { getSystemPhase } from '../narrative'`
2. **`src/components/screens/GameMain.tsx`**：
   - 添加 `lastTurnResult` ref 保存 `processTurn` 返回的真实 `TurnResult`
   - `handleChoice` 中使用保存的 `turnResult` 替代 `dummyTurnResult`，修复回合效果（经验、财富、掉落等）丢失的问题

### 相关发现
- `handleChoice` 之前使用 `dummyTurnResult`（所有 effect 为0），导致 `processTurn` 生成的经验、财富、道具掉落等效果完全丢失，玩家状态永远不会推进
- `updatePlayerAfterTurn` 只在 `handleChoice` 中调用，`startNewTurn` 不直接更新玩家状态，这是预期流程（先显示场景→玩家选择→应用效果→下一回合）
