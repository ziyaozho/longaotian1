人物绘图系统 — 风格方向重设计

当前问题：简单 Canvas 几何体拼出的人物（球体+圆柱体）与航海王式漫画风格差距太大

🎯 目标效果参考

用户想要的是 航海王 (One Piece) 式的漫画人物绘图 — 夸张比例、动态线条、表情丰富、有角色辨识度。不是简单的几何形状拼贴。

📸 对比：当前效果 vs 目标效果

❌ 当前效果（Canvas 几何体）

圆形头、椭圆眼睛、简单梯形身体
像乐高人偶，不像漫画人物

✅ 目标效果（航海王式漫画人物）

有锐利轮廓的下巴、漫画眼（瞳孔+高光）
飘逸的头发线条、肩部动态、服装褶皱

🔧 实现方案选择

哪种方案最能产出航海王式漫画人物？

A

SVG 精细绘制（推荐）

用复杂 SVG 路径替代简单几何体：刀削下巴、漫画眼（3层高光）、飘逸发束、动态褶皱

优点

  • 无外部依赖，纯前端
  • 可达到高质量漫画效果
  • 完全可控，支持任意属性组合
  • 零 API 成本

缺点

  • 开发工作量大（精细路径绘制）
  • 需要漫画人物绘画知识
B

AI 图像生成

通过 AI API 生成真正的漫画人物肖像，缓存后用作角色立绘

优点

  • 画质最高，真正的漫画画风
  • 开发量最小
  • 每次生成都是独特的

缺点

  • API 成本（每次几分钱）
  • 生成延迟（2-10秒）
  • 表情动态更新困难（需重新生成或叠加层）
C

混合方案：AI 底图 + SVG 动态层

AI 生成一张高品质底图，表情/装备/状态用 SVG 叠加绘制

优点

  • 底图质量最高
  • 表情仍可实时动态更新
  • 兼顾品质和交互性

缺点

  • 架构最复杂
  • 仍有 API 成本
  • 底图和叠加层可能风格不统一
