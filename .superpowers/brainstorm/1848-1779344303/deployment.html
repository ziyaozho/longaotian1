小程序对接方案
方案 1：WebView 容器（推荐，改动最小）
小程序内嵌 WebView → 加载 H5 页面 → 通过 JSBridge 通信
改动量：创建小程序壳工程（~50 行）+ JSBridge 适配层（~30 行）
⚠ 限制：部分 JS API 不可用，需测试
方案 2：原生小程序重写（改动大，不推荐）
用抖音小程序框架（TTML/TTSS/JS）重写 UI 层
改动量：全部 UI 组件需重写
⚠ 风险：工期不可控，不适用于黑客松

构建优化清单

代码分割

Demo 预缓存文件单独打包，正常模式不需要加载。动态 import 叙事内容文件。

移动端适配

添加 viewport 优化、触摸交互支持、安全区域适配。Demo 演示大概率在手机上扫码体验。

离线可用

Service Worker 缓存关键资源，离线模式下模板引擎完全可用（已有基础）。

二维码生成

收尾阶段展示游戏二维码，评委和观众可即时扫码体验。部署后自动生成。

推荐部署策略

1

主力：Vercel 自动部署

Git Push → 自动构建 → 自动部署。支持预览域名（PR 预览）和自定义域名。内置 HTTPS 和 CDN。

2

备用：CloudFlare Pages

如果需要国内访问速度更快的 CDN 节点。同样支持 Git 自动部署。

3

兜底：抖音小程序 WebView

一旦确认比赛要求小程序形态，1 天内可完成 WebView 壳 + JSBridge 适配。游戏核心零改动。
