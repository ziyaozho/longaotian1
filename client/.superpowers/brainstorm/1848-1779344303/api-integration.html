流式响应架构

SSE (Server-Sent Events) — 从"等全部生成完再显示"变成"边生成边显示"
📤 发送请求 → stream: true
↓ SSE 事件流
data: {"token":"天"}
data: {"token":"元"}
data: {"token":"大陆"}
data: {"token":"，"}
data: {"token":"灵气"}
...
↓ 逐字渲染到打字机组件
✅ 完成 → data: [DONE] → 显示选项

上下文窗口管理

如何在不超 token 上限的前提下携带尽可能多的游戏状态

结构化 JSON
