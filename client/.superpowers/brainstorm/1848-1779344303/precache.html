预缓存系统设计

Demo 模式下 AI 内容秒出，不依赖网络，但看起来和实时生成一模一样

工作原理

预缓存生命周期
① 构建时：运行 pnpm run precache 脚本
脚本读取 demoConfig 中的事件序列 → 逐个调用 DeepSeek API → 结果写入 precache/*.json
② 运行时（Demo 模式）：precacheService 直接从 JSON 读取
precacheService.lookup(eventId) → 返回预生成内容 → 模拟打字机动画输出
③ 运行时（正常模式）：走原有的 deepseek.ts 实时调用
完全不受影响，precache 层对正常模式透明

预缓存生成脚本

scripts/generate-precache.ts
// 伪代码示意
import { demoConfig } from '../src/demo/demoConfig';
import { callDeepSeek } from '../src/ai/deepseek';
import { writeFileSync } from 'fs';

async function generatePrecache() {
  const cache = { events: {} };

  for (const event of demoConfig.eventSequence) {
    console.log(`生成: ${event.id}...`);
    const result = await callDeepSeek(event.prompt);
    cache.events[event.id] = result;
  }

  writeFileSync('src/demo/precache/demo_v1.json', JSON.stringify(cache, null, 2));
  console.log('✅ 预缓存生成完成');
}

generatePrecache();

容错策略

缓存未命中

如果 eventId 在缓存中找不到 → 自动回退到实时 API 调用，同时记录 warning 日志

缓存过期

如果 demoConfig 版本号与缓存 version 不匹配 → 构建时报错，提示重新执行 precache 脚本

网络恢复

Demo 模式下即使有网络，也优先使用缓存。只有缓存损坏时才实时调用
